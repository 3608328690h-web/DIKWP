# Contributing / 贡献指南

贡献优先级：复现报告 > 新场景 > 失败案例 > 评分改进 > 新概念。

## Required for a new scenario

1. 一个对照样本和一个风险样本；
2. 同一业务目标、只改变一个关键因果变量；
3. 明确目的合同、权限、证据、预期门控和红线；
4. 不含真实个人数据、生产凭证或危险执行代码；
5. 添加单元测试和局限说明。

## Pull request rule

任何改变评分权重、红线或 Kill 条件的 PR 必须：
- 给出前后差分；
- 在全量 72 场景上重跑；
- 披露受益与受损的基线；
- 由至少两名维护者批准，其中一名应为非作者。
