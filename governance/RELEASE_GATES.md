# Release Gates

## G0 Identity and scope
- Project name, version and non-goals are explicit.
- No generated content is represented as an unapproved personal statement.

## G1 Data and licences
- Every scenario is synthetic and paired.
- Data and code licences are declared.

## G2 Reproducibility
- Scenario hash is stable.
- Baseline results reproduce on a clean Python environment.
- Tests pass.

## G3 Scientific quality
- Preregistration frozen before external submissions.
- Negative results and benchmark weaknesses can be reported.

## G4 Governance
- Scoring changes have public diffs and two reviews.
- Conflicts of interest are disclosed.

## G5 Release
- Git tag, release notes, checksums, citation metadata and archive are published.

## G6 External selection
- At least one non-author reproduction report is linked.
- A maintainer other than the founder can complete a release.
